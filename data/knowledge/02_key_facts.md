# 核心事实：Transformer 的关键工程与原理要点

## 事实 1：Self-Attention 是核心计算瓶颈之一

标准注意力在序列长度为 n 时，时间与显存复杂度近似为 O(n^2)。  
因此长上下文场景（如 32k、128k）会显著增加推理成本。  
常见优化方向包括 FlashAttention、分块注意力与稀疏注意力。

## 事实 2：预训练目标决定模型能力侧重点

典型目标：
1. Causal LM（下一词预测）偏向生成能力（GPT 类）
2. Masked LM（掩码恢复）偏向理解表征（BERT 类）
3. Seq2Seq（编码-解码）适合翻译、摘要等条件生成（T5 类）

## 事实 3：位置编码方案影响长文本外推能力

常见方案包括绝对位置编码、相对位置编码、RoPE 等。  
现代 LLM 普遍采用 RoPE 或变体，并通过插值/缩放提升长上下文表现。  
不合理的位置策略会导致长序列退化明显。

## 事实 4：训练稳定性依赖归一化与优化细节

关键实践：
1. 使用 LayerNorm/RMSNorm 保持深层训练稳定
2. 采用学习率预热 + 衰减策略
3. 使用梯度裁剪与混合精度训练降低数值风险
4. 在大批量训练中配合梯度累积与分布式并行

## 事实 5：推理优化对落地成本影响极大

常见策略：
1. KV Cache 降低自回归重复计算
2. 量化（INT8/INT4）降低显存与延迟
3. 连续批处理（Continuous Batching）提高吞吐
4. 推理框架优化（vLLM/TensorRT-LLM 等）提升效率
